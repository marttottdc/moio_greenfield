# Sprint Design Notes

This directory tracks design decisions for active workstreams so we do not lose context while implementation happens across branches.

Current sprint docs:

- `per-user-agents.md`: move from workspace-first runtime assumptions to per-user agent resolution while keeping workspaces as explicit shared collaboration spaces.
- `plugins.md`: introduce platform-managed capability bundles ("plugins") with strict readiness checks before they can appear in an agent's tool set.

These notes are intentionally product-facing and architecture-focused. They should be updated when scope, invariants, or rollout order changes.
